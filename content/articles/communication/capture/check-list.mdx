プロダクトの画面キャプチャや、その他ウェブサイトの画面キャプチャを作成する際、一貫性があり、わかりやすく、見栄えのよい画像を作成するための簡易チェックリストです。このリストに記載されている項目を満たすと、おおよそ作成ガイドラインにあげた品質を達成できます。  
チェックする内容によっては例外があることがあります。参考リンクも参照してください。



## 内容の調整

### 利用シーンに近い表現
- [ ] [氏名や日付など、実際に重複しない要素は表記を分散させている](./creation-guideline/)
- [ ] [特定の性別、年齢層、カテゴリに表記の偏りがない](./creation-guideline/)

### サンプル
- サンプルテキスト
    - [ ] [従業員氏名はユーザー社の従業員に掲載されている氏名を利用している](./creation-guideline/)
    - [ ] [メールアドレスは `{名前.苗字}@example.com` の形式になっている]()
    - [ ] [企業名は `株式会社＊＊＊＊＊` になっている]()
    - [ ] [従業員番号は `0064` など半端な数字を使用している]()
    - [ ] [住所、電話番号はデザインシステムで推奨する表記になっている]()
- サンプル画像
    - [ ] [プロフィール画像はモデル写真を使用している]()


## 見た目の調整

### ウィンドウサイズ
#### パソコンの場合
- 横幅
    - [ ] [横幅は1400pxで作成している]()
- 縦幅
    - [ ] [ページの最上部から最下部まで、全体を収めている]()
    - [ ] [縦幅は880px以上で作成している]()
    - [ ] [機能を象徴する重要な要素は、最上部から880px以内に収めている]()

#### スマートフォンの場合
- 横幅
    - [ ] [横幅は375pxで作成している]()
- 縦幅
    - [ ] [ページの最上部から最下部まで、全体を収めている]()
    - [ ] [縦幅は760px以上で作成している]()
    - [ ] [機能を象徴する重要な要素は、最上部から760px以内に収めている]()

### フォント
- [ ] [可読性が保たれた読みやすいウェイトになっている]()

### 色
- [ ] [視認性が保たれた読みやすい色になっている]()

### その他
- [ ] 画面のスクロールバーは非表示になっている
