    <tr>
      <Td>画面全体を俯瞰して捉えられない</Td>
      <Td>
        拡大鏡の拡大表示範囲外の変化を認識できない
        <ul>
          <li>
            1-1 ページ遷移したのか分からない： 
            リンクテキストをクリックした際に画面の範囲外で起こる変化に気づけないため、何のページに遷移したのか分からなくなる。
          </li>
          <li>
            1-2 通知表示に気づけない： 
            拡大鏡の拡大表示範囲外に表示される通知バナーなどに気づけない。
          </li>
          <li>
            1-3 UIの変化に気づけない： 
            ダイアログなど、ボタンを押した際に突如拡大鏡で見えている範囲全体を覆うようなUIが表示される場合、画面の変化を認識することが難しい。
          </li>
        </ul>
        拡大鏡の拡大表示範囲外の要素を認識できない・特定できない
        <ul>
          <li>
            1-4 要素に気づけない:
            画面の右端や動線から外れた場所にあるボタンなどに気づけない。
          </li>
          <li>
            1-5 目的の要素にたどり着くのが困難： 
            ユーザが拡大鏡の範囲外にある要素について認知していても、その要素にたどり着くまでに多くの時間がかかる場合がある。
          </li>
        </ul>
      </Td>
      <Td>
        <ul>
          <li>
            下線付きテキストなど、ページ遷移が発生することが分かるリンクテキストを設定する。（1-1）
          </li>
          <li>重要な通知は操作部の近くで表示する。（1-2）</li>
          <li>
            発生する画面の変化をユーザーへ事前に示唆する。UI表示後にユーザーに行って欲しい操作が分かるようなUIにする。（1-3）
          </li>
        </ul>
        <ul>
          <li>
            操作対象のすぐそばに操作部を配置する。たとえば表示の切り替えボタンは、切り替わる対象のすぐ近くに配置するようにする。（1-4）
          </li>
          <li>
            ページの主要な導線にUIを配置する。ページの上部と下部を行ったり来たりするような設計にしない。たとえば、フォームの入力完了ボタンがページの上部にあるような設計になっていると複数のフォームを入力した後に再度ページを上までスクロールしボタンを押さなければならない。（1-5）
          </li>
        </ul>
      </Td>
    </tr>
    <tr>
      <Td>視力が低い</Td>
      <Td>
        狙ったポインタ操作が困難:
        <ul>
          <li>
            2-1 クリックが困難： 小さな領域を狙ったマウスによるクリックが困難。
          </li>
          <li>
            2-2 ドラッグが困難： 
            ドラッグによるスクロールバーの操作や対象物の移動が困難。
          </li>
        </ul>
      </Td>
      <Td>
        <ul>
          <li>クリック： 操作可能な領域のサイズを大きくする。（2-1）</li>
          <li>
            ドラッグ（スクロール）:
            過度にスクロールが発生しないよう情報量に合わせたレイアウトを設計する。（2-2）
          </li>
          <li>
            ドラッグ（対象物の移動などスクロール以外）:
            クリック操作（キーボード操作）による代替手段を用意する。（2-2）
          </li>
        </ul>
      </Td>
    </tr>
    <tr>
      <Td>色の識別がしづらい</Td>
      <Td>
        <ul>
          <li>
            3-1 コントラスト比が低い要素に気づけない： 
            グレー色のテキストや画像、disabled状態の要素を知覚できない。
          </li>
          <li>
            3-2 背景色の違いに気づけない： 
            背景色の差によって提示された領域の区切りに気付けない。
            例：<a href="https://story.smarthr-ui.dev/?path=/docs/data-display%EF%BC%88%E3%83%87%E3%83%BC%E3%82%BF%E8%A1%A8%E7%A4%BA%EF%BC%89-basecolumn--docs">BaseColumnコンポーネント</a>
          </li>
          <li>
            3-3 表示色を設定している時、色の違いによって示している情報が無効可される： 
            設定により、要素の色が上書きされ、色の違いで示している情報は伝わらない。
            例：<a href="https://developer.mozilla.org/ja/docs/Web/CSS/@media/forced-colors">強制カラーモード</a>
          </li>
        </ul>
      </Td>
      <Td>
        <ul>
          <li>高いコントラスト比率を確保する（3-1）</li>
          <li>色だけで情報を提示しない（3-1, 3-2） </li>
          <li>システムカラーを活用する（3-3）</li>
        </ul>
      </Td>
    </tr>
  </tbody>
